# 7hats Human

Use for the human operating conditions that determine whether all other hats can function well.

## Load Order
- `references/lens.md`
- `references/diagnostics.md`
- `references/playbooks.md`
- `references/templates.md`

## When To Use
- Team truth-flow is blocked by fear, ambiguity, or unresolved tension.
- Decision ownership is unclear.
- Execution is degrading due to burnout or coordination friction.

## Required Inputs
- Team context and current operating pain points.
- Key decision/ownership breakdown points.
- Relevant collaboration signals (meeting friction, escalation churn, unresolved blockers).

## Workflow
1. Evaluate H-OS layers from `references/lens.md`.
2. Identify where truth mobility is blocked.
3. Define minimum repair actions (ownership, decision logs, communication cadence).
4. Apply team reset playbook when systemic friction persists.
5. Return request-scoped guidance with explicit owners and timelines.

## Output Contract
- Include:
  - ownership clarity
  - risk surfacing mechanism
  - team-health action plan
  - decision communication pattern
- Include `Source References`.
- Include `Readiness Verdict` when readiness is in question.

## Failure/Refinement Behavior
- Return `Needs Refinement` when:
  - no owner/timeline is defined
  - conflict/risk remains unnamed
  - plan relies on vague morale language without operational actions
- Include:
  - `Failed Gates`
  - `Missing Sources`
