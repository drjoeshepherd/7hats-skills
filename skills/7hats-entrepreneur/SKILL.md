# 7hats Entrepreneur

Use for portfolio trade-offs, bet sizing, and capital/time allocation decisions.

## Load Order
- `references/lens.md`
- `references/diagnostics.md`
- `references/playbooks.md`
- `references/templates.md`

## When To Use
- Competing initiatives require allocation trade-offs.
- Mission has large uncertainty and investment risk.
- Team needs explicit kill criteria before scaling effort.

## Required Inputs
- Candidate initiative(s).
- Resource constraints (time, capacity, budget).
- Decision horizon and review cadence.

## Workflow
1. Frame initiative as explicit bet.
2. Define minimum proof and measurement plan.
3. Define kill criteria and review date.
4. Check stage-gate readiness from `references/lens.md`.
5. Recommend exposure level (increase/hold/reduce) based on evidence.

## Output Contract
- Include:
  - hypothesis
  - minimum proof
  - kill criteria
  - review date
  - allocation implication
- Include `Source References`.
- Include `Readiness Verdict` when readiness is in question.

## Failure/Refinement Behavior
- Return `Needs Refinement` when:
  - no explicit hypothesis exists
  - minimum proof is not measurable
  - kill criteria or review date is missing
- Include:
  - `Failed Gates`
  - `Missing Sources`
