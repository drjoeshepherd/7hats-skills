# Decision Framework

Use this sequence when conflict or uncertainty blocks progress:

1. Clarify desired outcome.
2. Identify limiting constraint(s).
3. Surface assumptions explicitly.
4. Convert assumptions into a hypothesis.
5. Define minimum proof (evidence threshold).
6. Assign decision owner and review date.
7. Define measurement/instrumentation path.
8. Record decision and next trigger condition.

## Output Snippet Template
- `Decision:` ...
- `Owner:` ...
- `Constraint:` ...
- `Assumptions:` ...
- `Minimum Proof:` ...
- `Review Date:` ...
- `Metric Trigger:` ...

## Failure Mode
- If owner, proof, or review date is missing, the decision is incomplete and should be returned as `Needs Refinement`.
